# 3장. Node.js와 Git — 왜 둘 다 필요한가

> 5분 정도 걸립니다. 단 **이미 깔려있을 확률이 높습니다**. 먼저 확인부터 합시다.

---

## 3.1 두 도구의 역할

이 봇 프로젝트를 돌리려면 두 가지 프로그램이 컴퓨터에 있어야 합니다.

| 도구 | 역할 | 비유 |
|---|---|---|
| **Git(분산 버전 관리 시스템)** | 코드를 GitHub에서 받아오고, 수정 내역을 저장 | 사진 정리 앨범(내 컴퓨터). GitHub은 그 사진을 올리는 클라우드 |
| **Node.js(자바스크립트 런타임)** | 내 컴퓨터에서 봇을 직접 실행 (`npm run dev`) | "내 컴퓨터에서 JavaScript를 돌리는 발전기" |

### 3.1.1 둘 다 필수인 이유

서로 다른 일을 합니다.

- **Git만 있고 Node가 없으면** → 코드는 받았는데 실행이 안 됨
- **Node만 있고 Git이 없으면** → 코드를 받을 방법이 없음 (GitHub에서 압축 파일로 다운로드는 가능하지만 7장에서 push를 못 함)

> 💡 다른 수업에서 이미 설치하신 분들이 많습니다. **확인부터 하고, 없으면 그때 설치하시면 됩니다.**

---

## 3.2 시작하기 전 — "내 컴퓨터 망가지지 않나요?"

> 학생들이 가장 자주 무서워하는 질문입니다.

**안심하셔도 됩니다.** 이 장에서 쓰는 명령어는 전부 **읽기 전용(read-only)** 입니다.

- `git --version` → "Git 버전 알려줘" (아무것도 안 바꿈)
- `node --version` → "Node 버전 알려줘" (아무것도 안 바꿈)
- `npm --version` → "npm 버전 알려줘" (아무것도 안 바꿈)

또한 **재설치를 해도 기존 작업 폴더는 안 지워집니다.** Git/Node는 그냥 "도구"라서, 이미 만든 폴더나 파일과는 완전히 분리되어 있습니다. 마음 편하게 진행하세요.

> 📌 **[참고] "읽기 전용"이라는 약속**
>
> `--version`, `--help` 같이 `--`로 시작하는 옵션은 거의 대부분 정보를 보여주기만 합니다. 컴퓨터에 어떤 변경도 가하지 않습니다. 이 자습서에서 처음 만나는 모든 새 명령어는 안전한 것이라고 생각하셔도 됩니다.

---

## 3.3 CMD(Command Prompt, 명령 프롬프트) 창 열기

이 자습서는 **CMD**를 기준으로 설명합니다.

> 💡 PowerShell도 있지만, 학생들이 더 자주 쓰는 CMD로 진행합니다. PowerShell이 편하신 분은 그것도 OK (명령어가 거의 똑같습니다).

### 3.3.1 여는 방법

1. 키보드에서 **윈도우 키** 누르기
2. `cmd` 라고 입력
3. **명령 프롬프트** 클릭

검은색 창이 뜨면 성공:

```
Microsoft Windows [Version 10.0.xxxxx]
(c) Microsoft Corporation. All rights reserved.

C:\Users\본인이름>
```

---

## 3.4 Git 설치 확인

CMD 창에 입력:

```cmd
git --version
```

### 3.4.1 케이스 A — 이미 설치되어 있음 ✅

이런 출력이 나오면 끝. 다음 절(3.5)로 넘어가세요.

```
git version 2.45.1.windows.1
```

(버전 숫자는 달라도 됩니다. 2.x 이상이면 충분합니다.)

### 3.4.2 케이스 B — 설치되어 있지 않음 ❌

이런 메시지가 나오면 설치가 필요합니다:

```
'git'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는
배치 파일이 아닙니다.
```

#### 설치 방법

1. 브라우저에서 <https://git-scm.com/downloads> 열기
2. **Windows** 클릭 → 다운로드 자동 시작
3. 다운받은 `.exe` 파일 실행
4. 설치 옵션은 **전부 기본값(Next 연타)**으로 진행해도 OK
5. 설치 끝나면 **CMD 창을 닫았다가 다시 열기** (중요!)
6. 다시 `git --version` 입력 → 버전이 나오면 성공

> ⚠ **여기서 학생들이 자주 막힙니다**: 설치 후 기존 CMD 창에서 명령어를 그대로 치면 여전히 "찾을 수 없다"고 나옵니다. **반드시 CMD를 닫고 새로 열어야** 새 설치를 인식합니다.

> 📌 **[참고] 왜 CMD를 다시 열어야 하나?**
>
> CMD는 켜질 때 **환경 변수(PATH)**라는 목록을 한 번만 읽습니다. 설치 프로그램이 이 목록에 Git 위치를 추가하더라도, **이미 열려있던 CMD**는 옛 목록을 그대로 사용합니다. 새 CMD를 열어야 새 목록을 읽습니다.

## 3.5 Node.js와 npm(Node Package Manager) 설치 확인

CMD 창에 입력:

```cmd
node --version
```

그리고 추가로:

```cmd
npm --version
```

> 💡 **npm(Node Package Manager)**은 Node.js와 함께 자동으로 설치되는 "패키지 다운로더"입니다. 따로 설치할 필요가 없습니다. 5장에서 `npm install`로 봇이 필요로 하는 부품들을 받을 때 사용합니다.

### 3.5.1 케이스 A — 이미 설치되어 있음 ✅

이런 식으로 두 줄 다 나오면 끝:

```
v20.11.1
10.2.4
```

(Node는 18 이상, npm은 9 이상이면 충분합니다. 더 높아도 OK.)

### 3.5.2 케이스 B — 설치되어 있지 않음 ❌

이런 메시지가 나오면 설치가 필요합니다:

```
'node'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는
배치 파일이 아닙니다.
```

#### 설치 방법

1. 브라우저에서 <https://nodejs.org> 열기
2. **LTS(Long Term Support, 장기 지원)** 라고 적힌 큰 버튼 클릭 (예: "20.x.x LTS")
3. 다운받은 `.msi` 파일 실행
4. 설치 옵션은 **전부 기본값(Next 연타)**으로 진행
5. 마지막에 "Tools for Native Modules" 체크박스가 나오면 **체크 안 해도 됩니다** (그냥 Next)
6. 설치 끝나면 **CMD 창을 닫았다가 다시 열기** (중요!)
7. 다시 `node --version`, `npm --version` 입력 → 버전이 나오면 성공

> ⚠ **여기서도 학생들이 자주 막힙니다**: Git 때와 같은 이유입니다. 새 CMD 창에서 확인하세요.

---

## 3.6 확인하기

새 CMD 창을 열고, 아래 3줄을 차례대로 입력하여 **세 줄 모두 버전 숫자가 나오면 성공**:

```cmd
git --version
node --version
npm --version
```

기대 출력 (버전 숫자는 달라도 됨):

```
git version 2.45.1.windows.1
v20.11.1
10.2.4
```

### 3.6.1 검증 표

| 명령어 | 기대 결과 |
|---|---|
| `git --version` | `git version 2.x.x` 형식의 출력 |
| `node --version` | `v20.x` 또는 `v22.x` 형식 (18 이상이면 OK) |
| `npm --version` | `10.x` 또는 `9.x` 형식 |

세 줄 모두 ✅이면 4장으로.

## 3.7 🛟 자주 묻는 질문

**Q. 이미 설치되어 있는데, 최신 버전으로 다시 깔아야 하나요?**
> 아니요. 위의 최소 버전(Git 2.x, Node 18.x 이상)만 넘으면 충분합니다. 굳이 재설치하지 마세요.

**Q. 재설치하면 기존 코드가 지워지나요?**
> 안 지워집니다. Git/Node는 "도구"라서 작업 폴더(`C:\dev` 등)와 완전히 분리되어 있습니다. 마음 편하게 재설치해도 됩니다.

**Q. 설치 후에도 `git --version`이 안 나와요.**
> 99% 확률로 **CMD 창을 새로 안 연 것**이 원인입니다. 기존 CMD를 닫고 새 CMD를 열어 다시 시도하세요. 그래도 안 되면 컴퓨터 재부팅을 한 번 하면 거의 해결됩니다.

**Q. PowerShell에서도 같은 명령어를 쓰면 되나요?**
> 네, 똑같이 됩니다. PowerShell, CMD, Windows Terminal 어디서든 `git --version`, `node --version` 모두 작동합니다. 이 자습서는 CMD 기준입니다.

**Q. Mac/Linux 사용자입니다.**
> 동일한 명령어(`git --version`, `node --version`, `npm --version`)를 터미널에서 실행하면 됩니다.
> - Mac에 Git이 없으면 → 보통 `git --version` 첫 실행 시 Xcode Command Line Tools 설치 팝업이 뜹니다. 그거 설치하면 끝.
> - Node가 없으면 → [nodejs.org](https://nodejs.org) LTS 다운로드 (또는 `brew install node`)

**Q. 회사 PC라서 관리자 권한이 없어 설치를 못 합니다.**
> Git/Node 둘 다 **portable 버전**이 있긴 하지만, 처음이라면 학교/집 PC에서 진행하는 것을 추천합니다. 회사 PC에서 꼭 해야 한다면 IT 부서에 "Node.js LTS, Git for Windows 설치 요청"이라고 부탁해보세요.

**Q. Node 버전이 16인데 괜찮나요?**
> 가능은 하지만 권장하지 않습니다. 이 프로젝트는 Vite 8을 쓰는데, Vite 8은 Node 18+가 필요합니다. nodejs.org에서 LTS(20.x 이상)로 업데이트하시는 게 안전합니다.

## 3.8 이 장에서 배운 것 — 요약

| 항목 | 핵심 |
|---|---|
| Git 역할 | 코드 받아오기 + 수정 내역 저장 (GitHub과의 통로) |
| Node.js 역할 | 내 컴퓨터에서 자바스크립트 실행 |
| npm 역할 | Node 설치 시 자동으로 따라옴. 패키지(부품) 다운로더 |
| 설치 확인 | `git --version`, `node --version`, `npm --version` 세 줄 모두 OK |
| 설치 후 주의 | **반드시 새 CMD 창**에서 확인 (환경 변수 갱신 때문) |
