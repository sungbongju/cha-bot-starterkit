# 6장. 내 아바타 넣기

> 5분 정도 걸립니다. 봇에 얼굴을 만들어주는 단계입니다.

---

## 6.0 이 장에서 할 일

5장 마지막에 봇 화면에는 "👋 아바타가 비어있어요"라는 안내가 떴습니다. 이제 [1장](./03-chapter-1.md)에서 준비한 VRM(Virtual Reality Model) 파일을 적절한 위치에 넣어 캐릭터를 살립니다.

> **이 장에서 가장 헷갈리는 부분입니다.** 봉주도 처음에 잘못된 폴더에 넣어서 헤맸습니다. 그래서 **CMD 명령어로 넣는 방법**을 추천합니다 — 절대 실패하지 않는 방법입니다.

---

## 6.1 VRM 파일 구하기 (이미 준비되어 있으면 스킵)

[1.5절](./03-chapter-1.md#15-vrmvirtual-reality-model-파일-준비--약-530분)에서 이미 준비하셨다면 다음 절로 건너뛰셔도 됩니다. 다시 한번 요약:

### 6.1.1 방법 A. VRoid Hub에서 다운로드 ⭐

1. <https://hub.vroid.com> 접속
2. 마음에 드는 캐릭터 클릭
3. **다운로드** 가능한지 확인 (모든 캐릭터가 다운로드 가능한 건 아닙니다)
4. 다운로드 → `.vrm` 파일이 다운로드 폴더에 저장됨

### 6.1.2 방법 B. VRoid Studio로 직접 만들기

1. <https://vroid.com/studio>에서 VRoid Studio 다운로드 (무료)
2. 설치 후 실행 → 캐릭터 만들기
3. 처음이라면 **AvatarSample_A** 템플릿으로 시작하면 빠름
4. **Export → VRM**으로 내보내기

> ⚠ **License 경고 — VRoid Studio Export 시 반드시 확인하세요:**
>
> 기본값은 제약이 많습니다. 자신이 만든 캐릭터라도 다음과 같이 설정하시면 자유롭게 쓸 수 있습니다:
> - `Avatar Permission`: **everyone**
> - `Commercial Use`: **allow**
> - `Redistribution`: **allow**
> - `Modification`: **allow**

받은 `.vrm` 파일은 일단 **바탕화면**에 두세요. (이름이 예를 들어 `AvatarSample_A.vrm`이라고 가정)

## 6.2 `public/` 폴더에 `avatar.vrm`으로 넣기

### 6.2.1 ⭐ 방법 A — CMD 명령어 (추천, 절대 실패 안 함)

> 봉주가 시행착오 끝에 도달한 결론: **CMD가 가장 빠르고 확실합니다.** 파일 탐색기 드래그는 실수 가능성이 높습니다.

**새 CMD 창**을 엽니다 (기존 `npm run dev` 창은 그대로 두고 — [5.2.2](./08-chapter-5.md#522--매우-중요--이-cmd-창을-닫지-마세요) 주의사항).

다음 두 줄을 입력하세요:

```cmd
cd C:\dev\my-bot\public
copy "%USERPROFILE%\Desktop\AvatarSample_A.vrm" avatar.vrm
```

> 💡 `%USERPROFILE%\Desktop`은 "내 바탕화면"의 자동 경로입니다. `AvatarSample_A.vrm` 부분은 본인이 받은 파일 이름에 맞게 바꾸세요.

성공 시 출력:

```
        1개 파일이 복사되었습니다.
```

이게 핵심입니다: **파일 이름은 반드시 `avatar.vrm`** (소문자, 정확히 이 이름) — 봇이 이 이름을 찾도록 코딩되어 있습니다.

### 6.2.2 방법 B — 파일 탐색기 드래그앤드롭 (백업)

만약 위 CMD 방법이 안 되면 이걸 시도:

1. 파일 탐색기 열기 (윈도우 키 + E)
2. 주소창에 입력: `C:\dev\my-bot\public` (엔터)
3. 바탕화면에 있는 `.vrm` 파일을 이 폴더로 드래그
4. 복사된 파일을 우클릭 → 이름 바꾸기 → `avatar.vrm` (확장자 포함, 정확히)

> ⚠ **여기서 봉주도 헷갈렸습니다**: 파일 탐색기로 옮기다 보면 `public` 옆에 있는 다른 폴더 (`src` 등)에 실수로 떨어뜨리기 쉽습니다. **CMD 방법(6.2.1)이 훨씬 안전**합니다.

> 📌 **[참고] 왜 `public/` 폴더인가?**
>
> Vite(이 봇이 쓰는 빌드 도구)의 약속입니다. `public/` 폴더에 둔 파일은 **그대로 웹사이트의 루트(`/`)에 노출**됩니다. 즉 `public/avatar.vrm`은 브라우저에서 `http://localhost:5173/avatar.vrm`으로 접근 가능. 봇 코드가 이 경로를 찾도록 짜여 있습니다. `src/` 폴더에 넣으면 빌드 시 변환되어 다른 이름이 되므로 봇이 찾지 못합니다.

## 6.3 브라우저 강력 새로고침

브라우저로 돌아가 **강력 새로고침**:

- Windows: **Ctrl + Shift + R**
- Mac: **Cmd + Shift + R**

> ⚠ 일반 새로고침(F5)으로는 캐시 때문에 이전 화면이 그대로 보일 수 있습니다. **Ctrl + Shift + R**로 강제 새로고침해야 새 아바타 파일을 읽습니다.

### 6.3.1 성공 확인

**🎉 캐릭터가 등장하면 성공!**

- 자연스럽게 눈을 깜빡이고
- 가슴이 호흡하듯 위아래로 움직이고
- 가만히 서있는 idle(아이들) 애니메이션을 합니다

### 6.3.2 검증 표

| 항목 | 기대 결과 |
|---|---|
| `dir C:\dev\my-bot\public\avatar.vrm` | 파일 1개가 출력됨 |
| 첫 번째 CMD 창 (`npm run dev`) | 에러 없이 계속 실행 중 |
| 브라우저에서 Ctrl+Shift+R 후 | 캐릭터가 보이고 움직임 |

## 6.4 🛟 자주 묻는 질문

**Q. 아바타를 넣었는데 안 보여요. 빈 화면이거나 안내문이 그대로예요.**
> 체크리스트:
> 1. 파일 이름이 정확히 `avatar.vrm`인가요? (대소문자 구분, `Avatar.vrm`도 안 됨)
> 2. 위치가 정확히 `C:\dev\my-bot\public\avatar.vrm`인가요? (`src/` 폴더에 넣으면 안 됩니다)
> 3. 브라우저에서 **Ctrl+Shift+R** 강제 새로고침 했나요?
>
> CMD에서 확인:
> ```cmd
> dir C:\dev\my-bot\public\avatar.vrm
> ```
> 파일이 보이면 위치는 OK.

**Q. 캐릭터가 너무 크거나 작아요. 위치도 이상해요.**
> 아바타마다 기본 크기가 달라서 그럴 수 있습니다. 2차/3차 자습서에서 카메라 위치·줌 조정 방법을 다룰 예정. 일단은 "보이기만 하면" 성공입니다.

**Q. VRM 파일을 어디서 받으면 좋을지 더 추천해주세요.**
> - [VRoid Hub](https://hub.vroid.com) — 가장 큰 라이브러리
> - [Booth](https://booth.pm) — 유료/무료 캐릭터 마켓 (일본 사이트)
> - VRoid Studio로 직접 만들기 — 완전한 본인 캐릭터를 원한다면 ⭐ 추천

**Q. 다른 사람이 만든 VRM을 봇에 써도 되나요?**
> **반드시 라이선스 확인이 필요합니다.** 캐릭터 만든 사람이 "타인 사용 허용", "상업적 이용 허용" 등을 명시했는지 봐야 합니다. VRoid Hub에서는 각 캐릭터 페이지 하단에 라이선스 표시가 있습니다.

**Q. Mac/Linux 사용자입니다.**
> 명령어 거의 동일합니다. 파일 복사 부분만 다음과 같이 바꾸세요:
> ```bash
> cd ~/dev/my-bot/public
> cp ~/Desktop/AvatarSample_A.vrm avatar.vrm
> ```

**Q. 봉주처럼 잘못된 폴더에 넣어서 헷갈렸을 때 어떻게 정리하나요?**
> 잘못 넣은 파일을 그 폴더에서 삭제하고, 위의 CMD 방법으로 다시 시도하세요. 잘못된 파일이 남아있어도 봇 동작에 영향은 없지만, 폴더가 어지러워지면 나중에 헷갈리므로 정리하는 게 좋습니다.

## 6.5 이 장에서 배운 것 — 요약

| 항목 | 핵심 |
|---|---|
| 넣을 위치 | `C:\dev\my-bot\public\avatar.vrm` (정확히 이 경로 + 이름) |
| 추천 방법 | CMD `copy` 명령어 (드래그 실수 방지) |
| 봉주 함정 ① | 드래그하다 `src/` 폴더에 떨어뜨림 → CMD 방법 권장 |
| 봉주 함정 ② | F5만 누르면 캐시 때문에 안 보임 → Ctrl+Shift+R 강력 새로고침 |
| `public/` 이유 | Vite의 규칙 — `public/`은 웹 루트에 그대로 노출됨 |
