# 서장. 왜 자기 봇을 만드는가

> 본격적인 실습 전에 큰 그림 먼저. 코드 한 줄도 치지 않습니다. 10분이면 읽으실 수 있습니다.

---

## 0.1 박대근 교수님의 비전 — 스타터킷에서 1인 창업 플랫폼까지

이 자습서가 다루는 `cha-bot-starter-kit`(차봇 스타터킷)은 **단순한 학습용 예제가 아닙니다**. 박대근 교수님께서 학생들에게 강조해 오신 비전의 첫 단추입니다.

### 0.1.1 박교수님의 시각 — "쉽게, 모두가, 자기 봇을"

박교수님께서 여러 자리에서 말씀하신 핵심 메시지는 다음 세 가지로 요약할 수 있습니다.

1. **쉽게 가르치자.** 학생이 한 번 막히면 영영 안 돌아옵니다. 모든 단계는 "혼자서도 따라할 수 있게" 만들어야 합니다.
2. **모두가 자기 봇을 가지자.** AI 봇은 일부 전공자만의 영역이 아니라, 누구든 자기 주제(학과 안내, 상품 추천, 게임, 학습 도우미 등)로 만들 수 있는 도구여야 합니다.
3. **스타터킷에서 끝나면 안 됩니다.** 학생이 봇을 띄운 뒤, 그것을 **자기 사업의 출발점**으로 삼을 수 있도록 — 즉 1인 창업 플랫폼으로 연결되어야 합니다.

### 0.1.2 그래서 이 스타터킷은 무엇을 합니다

- **VRM(Virtual Reality Model, 가상현실 모델) 아바타**가 화면에 떠서 눈을 깜빡이고 호흡합니다.
- **카메라·음성·텍스트** 세 가지 모드를 토글할 수 있습니다.
- **다크/라이트 테마**를 즉시 전환할 수 있습니다.
- 코드는 **GitHub**에서 받아 **Vercel**로 배포 — 전 세계 누구나 접속 가능한 URL이 생성됩니다.

이 모든 것을 **2시간 안에** 입문자가 직접 만들 수 있게 한 것이 이 자습서의 목표입니다.

> 📌 **[참고] "왜 VRM인가요?"**
>
> VRM은 일본의 픽시브(Pixiv)에서 시작된 3D 캐릭터 표준 포맷입니다. `.vrm` 파일 하나에 모델·뼈대·표정·라이선스 정보가 모두 들어 있어, 웹에서 바로 띄우기 쉽습니다. VRoid Hub·VRoid Studio 같은 무료 생태계가 잘 갖춰져 있어, 학생이 자기 캐릭터를 만들기에 가장 진입장벽이 낮습니다.

## 0.2 이 자습서의 위치 — 1차 / 2차 / 3차 자습서 지도

봉주가 작성 중인 전체 자습서는 **3개의 차수**로 구성됩니다. 여러분이 지금 보고 계신 것은 그중 **1차**입니다.

### 0.2.1 전체 지도

| 차수 | 제목 | 핵심 질문 | 결과물 | 상태 |
|---|---|---|---|---|
| **1차** | 봇을 인터넷에 띄우기까지 | "어떻게 하면 친구한테 URL을 보낼 수 있나?" | `https://my-bot.vercel.app` 라이브 사이트 | **이 책** ⭐ |
| 2차 | 내 봇으로 커스터마이즈 | "어떻게 내 캐릭터·내 주제로 바꾸나?" | 본인만의 VRM + 본인 주제 RAG(Retrieval-Augmented Generation) | 작성 예정 |
| 3차 | Claude Code로 자유 개조 | "어떻게 새 기능을 추가하나?" | 색상·배경·새 모드·게임 요소 추가 | 작성 예정 |

### 0.2.2 1차 자습서의 약속

1차 자습서를 마치시면 다음 세 가지를 손에 쥐게 됩니다.

- ✅ **로컬 개발 환경** — 내 컴퓨터에서 봇을 띄울 수 있는 셋업
- ✅ **GitHub 협업의 기초** — `clone → add → commit → push`의 흐름
- ✅ **인터넷에 실제 서비스 배포** — Vercel을 통한 자동 배포 파이프라인

이 세 가지는 봇뿐 아니라 **모든 웹 프로젝트의 표준 워크플로우**입니다. 한 번 익혀두면 평생 씁니다.

---

## 0.3 6단계 학습 로드맵

1차 자습서는 8개 장으로 나뉘지만, 큰 흐름은 **6단계**입니다.

```
[1] 준비물 갖추기        →  1장 (계정·도구·VRM)
       ↓
[2] 작업 공간 만들기      →  2장 (C:\dev)
       ↓
[3] 도구 확인           →  3장 (Node·Git)
       ↓
[4] 코드 받아오기        →  4장 (git clone)
       ↓
[5] 로컬에서 띄우기 + 아바타 추가  →  5·6장 (npm install / dev / avatar)
       ↓
[6] 인터넷에 배포        →  7·8장 (git push + Vercel)
```

각 단계는 **이전 단계의 결과물을 가정**합니다. 따라서 순서대로 진행하셔야 매끄럽습니다. 건너뛰면 거의 반드시 막힙니다.

---

## 0.4 박교수님 카톡 Step 1~6과 이 자습서의 매핑

박교수님께서 학생들에게 카톡으로 보내신 **Step 1~6 안내**가 있습니다. 그것이 이 자습서에서 어떻게 분배되는지 표로 정리합니다.

| 박교수님 Step | 카톡 메시지 요약 | 본 자습서 |
|---|---|---|
| Step 1 | GitHub 계정 만들기, VS Code/Node/Git 설치 | 1·2·3장 |
| Step 2 | 스타터킷 `git clone` | 4장 |
| Step 3 | `npm install`, `npm run dev`로 로컬 실행 | 5장 |
| Step 4 | `public/avatar.vrm` 추가 | 6장 |
| Step 5 | 본인 GitHub 새 레포 생성 후 push | 7장 |
| Step 6 | Vercel에 import → 배포 | 8장 |

> 📌 **[참고] "포크는 하지 맙시다"**
>
> 박교수님께서 명확히 정하신 방침입니다. GitHub의 fork(원본을 복사해 내 계정에 가져오는 기능) 대신, **`git clone` 받아서 새 레포를 만들고 그쪽으로 push**합니다. 이유는 fork가 원본과 영구 연결되어 나중에 충돌·동기화 이슈가 복잡해지기 때문. 7장에서 자세히 다룹니다.

## 0.5 시간 / 비용 / 결과물 예상

### 0.5.1 시간

| 구간 | 예상 시간 | 비고 |
|---|---|---|
| 1장 — 준비물 (계정·도구·VRM) | 15~20분 | 다운로드 대기 포함 |
| 2장 — `C:\dev` 폴더 | 5분 | |
| 3장 — Node·Git 확인 | 5~15분 | 이미 깔려있으면 5분 |
| 4장 — `git clone` | 5분 | |
| 5장 — `npm install` + `npm run dev` | 10~15분 | `npm install` 대기 포함 |
| 6장 — 아바타 추가 | 5분 | |
| 7장 — GitHub push | 15분 | 처음 인증 화면 포함 |
| 8장 — Vercel 배포 | 10분 | 빌드 대기 포함 |
| **총 합계** | **약 60~90분** | 첫 도전 기준 |

이미 도구가 깔린 분은 30~40분이면 완주 가능합니다.

### 0.5.2 비용

**전부 무료입니다.**

- GitHub — 무료 (개인 계정)
- Vercel — Hobby 플랜 무료 (월 100GB 대역폭, 학생/교수님 개인 용도로 충분)
- Node.js·Git — 오픈소스 무료
- VRM 파일 — VRoid Hub에서 무료 다운로드 가능
- 도메인(예: `my-bot.vercel.app`) — Vercel이 자동 무료 제공

본인 도메인(예: `mybot.com`)을 사고 싶을 때만 별도 비용(연 1만 원대~).

### 0.5.3 결과물

1차 자습서가 끝나면 여러분이 **갖게 될 것**:

1. **인터넷에 떠 있는 본인 봇 URL** — 예: `https://my-vrm-bot-xxxxxxxx.vercel.app`
2. **본인 GitHub에 올라간 봇 코드 레포** — 예: `https://github.com/내ID/my-bot`
3. **로컬 개발 환경** — `C:\dev\my-bot`에서 언제든 켜고 끌 수 있는 봇 서버
4. **자신감** — "나도 코드 프로젝트를 인터넷에 띄울 수 있구나"

특히 마지막 항목이 가장 큽니다. 코드 세계 입문의 두려움을 한 번 깨고 나면, 다음 프로젝트는 훨씬 가볍게 시작할 수 있습니다.
