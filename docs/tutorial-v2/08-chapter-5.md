# 5장. 로컬에서 봇 띄우기 — `npm install`과 `npm run dev`

> 10~15분 걸립니다. 이 장이 끝나면 **내 컴퓨터 브라우저에서 봇 화면**을 만나게 됩니다.

---

## 5.0 이 장에서 할 일

지금까지는 코드만 받았습니다. 이제 **실행**해 볼 차례입니다.

1. 봇 실행에 필요한 부품들 다운로드 (`npm install`)
2. 봇 서버 켜기 (`npm run dev`)
3. 브라우저로 접속해 봇 화면 보기

(아바타 추가는 [6장](./09-chapter-6.md)에서 별도로 다룹니다.)

---

## 5.1 부품 다운로드 — `npm install`

### 5.1.1 무엇을 하는가

CMD를 열고 봇 폴더로 이동:

```cmd
cd C:\dev\my-bot
```

이어서 다음 명령어 입력:

```cmd
npm install
```

### 5.1.2 비유 — 가구 조립 키트의 부품 봉투

> `npm install`은 IKEA 가구 키트로 비유할 수 있습니다.
>
> - `package.json` = 조립 설명서에 적힌 **부품 목록**
> - `npm install` = 그 목록을 보고 부품 봉투를 **모두 가져와 풀어놓는 일**
> - `node_modules/` 폴더 = 부품들이 깔린 작업대

약 300개 이상의 부품이 다운로드됩니다.

### 5.1.3 ⚠ 여기서 학생들이 가장 많이 막힙니다

> **1~3분 동안 화면이 안 움직이는 것처럼 보일 수 있습니다.** 멈춘 것 같지만 정상입니다. 인터넷에서 부품들을 받고 있습니다.

```
npm warn deprecated ...
npm warn deprecated ...
(한참 멈춤... 1분~3분)
```

> 🛟 **절대 Ctrl+C를 누르지 마세요.** 그냥 기다리세요. 봉주도 처음에는 "멈춘 줄 알고" Ctrl+C 눌렀다가 처음부터 다시 했습니다.

### 5.1.4 성공 시 출력

성공 시 마지막에 이런 출력이 나옵니다:

```
added 387 packages, and audited 388 packages in 2m

87 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities
```

(숫자는 다를 수 있고, "vulnerabilities" 경고가 몇 개 떠도 무시해도 OK.)

이제 `dir` 해보면 `node_modules`라는 새 폴더가 생긴 것을 볼 수 있습니다. 이게 다운받은 부품들입니다.

> 📌 **[참고] `node_modules`가 거대한 이유**
>
> 보통 200~500MB 크기입니다. 부품(라이브러리)들이 자기 부품을 또 가지고 있고, 그게 또 부품을... 식으로 의존성이 깊기 때문입니다. 이 폴더는 GitHub에 올리지 않습니다 — `.gitignore`로 자동 제외되며, 다른 사람이 코드를 받으면 본인 컴퓨터에서 `npm install`로 새로 받습니다.

---

## 5.2 봇 서버 켜기 — `npm run dev`

### 5.2.1 명령어 실행

같은 CMD 창에서 이어서 입력:

```cmd
npm run dev
```

몇 초 후 이런 출력이 나옵니다:

```
> cha-bot-starter-kit@1.0.0 dev
> vite

  VITE v8.0.3  ready in 1234 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```

**`http://localhost:5173/`가 보이면 성공!** 봇이 내 컴퓨터 안에서 돌아가고 있다는 뜻입니다.

### 5.2.2 ⚠⚠ 매우 중요 — 이 CMD 창을 **닫지 마세요**

> 봉주도 여기서 한 번 실수했습니다.

이 CMD 창은 **봇 서버가 살아있는 동안 계속 켜둬야** 합니다. 닫으면 봇도 꺼집니다.

**또 다른 명령어를 쳐야 한다면 → 새 CMD 창을 따로 여세요.** (6장과 7장에서 그렇게 합니다.)

> 📌 **[참고] localhost(로컬호스트)란?**
>
> `localhost`는 "내 컴퓨터 자기 자신"을 가리키는 특별한 주소입니다. `http://localhost:5173`은 "내 컴퓨터의 5173 포트(port, 통신 창구 번호)에 접속"이라는 뜻. 외부에서는 이 주소로 접속할 수 없습니다 — 인터넷에 띄우는 건 [8장](./11-chapter-8.md)에서.

---

## 5.3 브라우저로 접속

브라우저(Chrome, Edge, Firefox 무엇이든) 주소창에 입력:

```
http://localhost:5173
```

이런 화면이 나오면 성공입니다:

```
┌──────────────────────────────────────┐
│                                      │
│         👋 아바타가 비어있어요         │
│                                      │
│    public/avatar.vrm 파일을 추가하면   │
│      여기에 캐릭터가 나타납니다.        │
│                                      │
└──────────────────────────────────────┘
```

> 💡 이 친절한 안내문은 "아직 아바타 파일이 없다"는 뜻입니다. [6장](./09-chapter-6.md)에서 추가하면 사라지고 진짜 캐릭터가 등장합니다.

### 5.3.1 화면이 안 나올 때

- **빈 화면**: 페이지 새로고침 (F5) 한 번 해보세요.
- **"이 사이트에 연결할 수 없음"**: CMD 창에서 `npm run dev` 출력이 정상인지 다시 확인. 에러 메시지가 있다면 캡처해서 박교수님이나 봉주에게 보여주세요.
- **포트 충돌 에러** (`Port 5173 is already in use`): 다른 프로그램이 같은 포트를 쓰고 있습니다. CMD에서 Ctrl+C로 끄고 다시 `npm run dev` 하면 보통 다른 포트(5174 등)로 자동 변경됩니다.

---

## 5.4 종료와 재시작

오늘 작업을 끝내고 컴퓨터를 끄거나, 잠깐 쉬려면:

### 5.4.1 끄는 법

1. `npm run dev`가 돌아가는 CMD 창에서 **Ctrl + C** 누르기
2. "일괄 작업을 끝내시겠습니까 (Y/N)?" 물어보면 **Y** 입력 → 엔터
3. CMD 창 닫기

### 5.4.2 다음에 다시 시작할 때

```cmd
cd C:\dev\my-bot
npm run dev
```

두 줄만 다시 치면 됩니다. `npm install`은 한 번만 하면 됩니다 — 부품들이 이미 `node_modules`에 있기 때문.

### 5.4.3 검증 표

| 항목 | 기대 결과 |
|---|---|
| CMD에 `npm install` | 마지막에 `added N packages` 출력, 에러 없음 |
| CMD에 `npm run dev` | `VITE v8.x.x ready in ...` 줄 + `Local: http://localhost:5173/` |
| `dir` 출력 | `node_modules` 폴더가 새로 보임 |
| 브라우저에서 `http://localhost:5173` | "아바타가 비어있어요" 안내 화면 표시 |

## 5.5 🛟 자주 묻는 질문

**Q. `npm install`이 한참 멈춰있어요. 정말 정상인가요?**
> 네, 매우 흔한 일입니다. 인터넷 속도와 컴퓨터 성능에 따라 1~5분 걸립니다. **Ctrl+C 누르지 말고 그냥 기다리세요.** 5분 넘어가도 응답이 없으면 그때 Ctrl+C 후 다시 `npm install`.

**Q. `npm run dev` 출력에 빨간 글씨로 경고가 떠요. 망한 건가요?**
> "Warning"은 무시해도 됩니다. "Error"라고 적힌 빨간 글씨가 아니라면 정상입니다. `Local: http://localhost:5173/` 줄만 나오면 성공.

**Q. localhost:5173이 아닌 다른 포트로 나와요 (예: 5174).**
> 정상입니다. 5173이 이미 다른 프로그램에 의해 점유되어 있으면 자동으로 다음 번호로 바뀝니다. CMD에 출력된 그 주소를 그대로 브라우저에 입력하세요.

**Q. CMD 창을 두 개 띄우면 좋다고 하셨는데, 한 개만 쓰면 안 되나요?**
> 됩니다. 단 `npm run dev`가 도는 동안 그 창에서는 다른 명령어를 칠 수 없습니다. 다른 명령어가 필요한 순간(예: 6장의 `copy`, 7장의 `git add`)이 되면 새 창을 여는 게 깔끔합니다.

**Q. `node_modules` 폴더를 실수로 지웠어요.**
> 다시 `npm install` 하시면 됩니다. 약 1~3분 다시 기다리시면 복구.

**Q. Mac/Linux 사용자입니다.**
> 명령어 거의 동일합니다:
> ```bash
> cd ~/dev/my-bot
> npm install
> npm run dev
> ```

## 5.6 이 장에서 배운 것 — 요약

| 항목 | 핵심 |
|---|---|
| `npm install` | `package.json`에 적힌 부품을 모두 다운로드 → `node_modules/`에 저장 |
| 대기 함정 | 1~3분 멈춘 듯 보여도 정상. Ctrl+C 누르지 말 것 |
| `npm run dev` | 로컬 봇 서버 시작. `Local: http://localhost:5173/`이 나오면 성공 |
| CMD 창 주의 | `npm run dev` 창은 닫지 말 것. 다른 일은 새 창에서 |
| 종료 | Ctrl+C → Y → 엔터 |
| 재시작 | `cd C:\dev\my-bot` 후 `npm run dev` 한 줄이면 OK |
