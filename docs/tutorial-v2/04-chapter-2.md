# 2장. 작업 공간 만들기 — `C:\dev`

> 두 번째 장입니다. 5분 정도 걸립니다. 화면을 몇 번 클릭하면 끝납니다.

---

## 2.1 왜 `C:\dev` 폴더부터 만드는가

코드 프로젝트는 **짧고 깔끔한 폴더 경로**에 두는 것이 편합니다.

| 좋은 예 | 나쁜 예 |
|---|---|
| `C:\dev\my-bot\` ✅ | `C:\Users\홍길동\Documents\내 문서\프로젝트 모음\봇만들기\` ❌ |

### 2.1.1 짧은 경로를 선호하는 이유

1. **한글이 들어있으면 명령어가 가끔 깨집니다.** 일부 도구가 한글 파일명·경로를 제대로 처리하지 못합니다.
2. **띄어쓰기가 있으면 명령어 입력 시 따옴표를 일일이 넣어야 합니다.** `cd "C:\Program Files\..."` 같은 식.
3. **경로가 길면 Windows가 "경로가 너무 길어요" 에러를 냅니다.** 윈도우의 경로 길이 제한(보통 260자).
4. `C:\dev` 같이 **루트(C:) 바로 아래 짧은 영어 이름**이 가장 안전합니다.

### 2.1.2 비유 — 책상 위 vs 서랍 속 깊은 곳

```
[좋은 예: C:\dev\]
책상 위. 손 뻗으면 바로 닿음.

[나쁜 예: C:\Users\...\Documents\...\프로젝트 모음\...\]
서랍 속 → 서랍 속 폴더 → 다시 그 안의 봉투 → 그 안의 종이...
꺼낼 때마다 한참 걸림.
```

그래서 모든 코드 프로젝트는 `C:\dev\` 폴더 안에 모아두는 것이 관례입니다.

---

## 2.2 만드는 3가지 방법 — 택1

### 2.2.1 방법 A. 파일 탐색기로 (가장 쉬움) ⭐ 추천

1. 키보드에서 **윈도우 키 + E** → 파일 탐색기가 열립니다
2. 좌측 패널에서 **내 PC** 클릭 → **로컬 디스크 (C:)** 더블클릭
3. 빈 공간 우클릭 → **새로 만들기 → 폴더**
4. 폴더 이름을 `dev`라고 입력 → 엔터

→ 완료. 이제 `C:\dev` 폴더가 생겼습니다.

### 2.2.2 방법 B. PowerShell 명령어 (한 줄)

1. 시작 메뉴 → **PowerShell** 검색 → 실행
2. 입력:

   ```powershell
   mkdir C:\dev
   ```

3. 엔터

> 💡 이미 폴더가 있으면 "이미 존재합니다" 에러가 뜨는데 무시해도 됩니다. 폴더가 있으면 OK.

### 2.2.3 방법 C. CMD(Command Prompt) 명령어

1. 시작 메뉴 → `cmd` 검색 → **명령 프롬프트** 실행
2. 입력:

   ```cmd
   md C:\dev
   ```

3. 엔터

`md`는 "make directory(디렉터리 만들기)"의 줄임말입니다.

---

## 2.3 확인하기

파일 탐색기 주소창에 `C:\dev` 입력 후 엔터 → 빈 폴더가 열리면 성공.

또는 PowerShell/CMD에서:

```cmd
dir C:\
```

출력된 목록에 `dev` 폴더가 보이면 됩니다.

### 2.3.1 검증 표

| 항목 | 기대 결과 |
|---|---|
| 파일 탐색기 주소창에 `C:\dev` 입력 | 빈 폴더가 열림 |
| CMD에서 `dir C:\` 실행 | 출력 목록에 `dev` 폴더가 있음 |
| CMD에서 `cd C:\dev` 실행 | 프롬프트가 `C:\dev>`로 변경됨 |

---

## 2.4 🛟 자주 묻는 질문

**Q. 회사/학교 PC라서 C 드라이브에 폴더를 못 만들어요.**
> `D:\dev` 또는 `C:\Users\본인이름\dev`도 OK. 짧고 한글/공백이 없으면 됩니다. 이후 자습서에 나오는 `C:\dev` 부분을 본인 경로로 바꿔 읽으시면 됩니다.

**Q. 이미 다른 곳에 코드 폴더가 있어요.**
> 그대로 쓰셔도 됩니다. 단 이 자습서의 모든 `C:\dev\` 경로를 본인 경로로 머릿속에서 바꿔주세요.

**Q. Mac/Linux 사용자입니다.**
> `~/dev` 폴더를 만드세요 (`mkdir ~/dev`). 이 자습서의 `C:\dev`는 `~/dev`로 읽어주시면 됩니다.

**Q. 폴더 이름이 꼭 `dev`여야 하나요?**
> 아니요. `code`, `projects`, `work` 등 짧은 영어면 OK. 이 자습서는 관례인 `dev`로 진행합니다.

## 2.5 이 장에서 배운 것 — 요약

| 항목 | 핵심 |
|---|---|
| 작업 폴더 위치 | `C:\dev` (짧고, 한글·공백 없는 영어 경로) |
| 만드는 방법 | 파일 탐색기 / PowerShell `mkdir` / CMD `md` 중 택1 |
| 확인 명령어 | `dir C:\` 출력에 `dev`가 보이면 OK |
