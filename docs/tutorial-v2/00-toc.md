# [참고] 자습서 사용법

---

## 각 부의 역할

이 1차 자습서는 두 개의 부(Part)와 한 개의 서장(Chapter 0)으로 구성됩니다.

### 서장 — 큰 그림

본격적인 실습 전에 **왜 이걸 만드는가, 어디까지 만들 것인가, 박교수님의 비전은 무엇인가**를 정리합니다. 코드를 한 줄도 치지 않습니다. **10분이면 읽으실 수 있습니다.**

### 제1부. 준비물과 도구

봇을 만들기 위해 **사전에 갖춰야 할 도구·계정·파일**을 준비합니다. 한 번 갖추고 나면 이후 어떤 코드 프로젝트에서도 재활용할 수 있는 기본기입니다. **30분 정도** 소요.

- 1장 — GitHub·Vercel 계정, 도구 다운로드, VRM 파일
- 2장 — `C:\dev` 폴더 만들기
- 3장 — Node.js와 Git 설치 확인

### 제2부. 봇을 인터넷에 띄우기

본격적인 실습입니다. **스타터킷을 받아서 → 내 컴퓨터에서 띄우고 → 내 GitHub에 올리고 → 인터넷에 배포**합니다. 박교수님 카톡의 Step 1~6 흐름을 그대로 따라갑니다. **60분 정도** 소요.

- 4장 — `git clone`
- 5장 — `npm install`과 `npm run dev`
- 6장 — 내 아바타 넣기
- 7장 — `git push`
- 8장 — Vercel 배포

### 에필로그 — 완주 축하와 다음 단계

1차 자습서를 마치신 분께 박수와 함께, 2차/3차 자습서가 무엇을 다루는지 안내합니다.

## 처음 보는 분 vs 이미 해본 분

### 처음 보는 분 (대부분의 학생)

**서장부터 차례대로** 읽으세요. 건너뛰지 마시기를 권합니다. 특히:

- **1장(준비물)** — 여기서 계정 가입과 다운로드를 미리 해두면 본 실습이 매끄럽습니다.
- **3장(Node·Git 확인)** — `git --version` 같은 명령어가 동작하지 않으면 4장 이후가 모두 막힙니다.

### 이미 Git·Node를 다뤄본 분

다음 순서를 권합니다:

1. **서장 0.1, 0.4만** 빠르게 읽고 → 이 봇이 어떤 봇인지 파악
2. **3장 끝의 검증 표**만 보고 본인 환경 점검
3. **4장(`git clone`)부터** 본격 시작

소요 시간 약 30분.

### Vercel 배포 경험이 있는 분

8장은 거의 새로운 내용이 없습니다. 7장까지만 따라하시고 곧장 [에필로그](./12-epilogue.md)로 가셔서 **2차/3차 자습서**를 확인하세요.

## 코드 블록 약속

본 자습서에는 두 종류의 코드 블록이 등장합니다.

### 입력 — 여러분이 직접 쳐야 하는 것

````
```cmd
git --version
```
````

CMD(Command Prompt) 창에 그대로 복사·붙여넣기 하시면 됩니다.

### 출력 — 컴퓨터가 보여주는 화면

````
```
git version 2.45.1.windows.1
```
````

이 출력이 보이면 정상입니다. 다르게 나오면 [자주 묻는 질문]을 확인하세요.
