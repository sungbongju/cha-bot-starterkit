# 제2부. 봇을 인터넷에 띄우기

> 이제 본격 실습입니다. 스타터킷을 받아 내 컴퓨터에서 띄우고, 내 GitHub에 올리고, 인터넷에 배포합니다.

---

## 제2부의 목표

박교수님 카톡 Step 2~6 흐름을 그대로 따라갑니다. 마지막에는 **친구에게 보낼 수 있는 인터넷 URL**이 손에 들어옵니다.

```
[스타터킷]               [내 컴퓨터]              [내 GitHub]              [인터넷]
github.com/...    →    C:\dev\my-bot     →    github.com/내ID/...    →    https://my-bot
                                                                              .vercel.app
   git clone           npm install              git push                 Vercel 자동 배포
   (4장)              npm run dev               (7장)                    (8장)
                      (5·6장)
```

---

## 제2부의 구성

```
4장. 스타터킷 받아오기 (git clone)
  └─ GitHub에서 코드를 내 컴퓨터로 복사 (5분)

5장. 로컬에서 봇 띄우기 (npm install + npm run dev)
  └─ 부품 다운로드 후 봇 서버 시작 (10~15분)

6장. 내 아바타 넣기
  └─ public/avatar.vrm 추가 + 브라우저 새로고침 (5분)

7장. GitHub에 내 봇 올리기 (git push)
  └─ 새 레포 생성 후 commit → push (15분)

8장. Vercel로 인터넷 배포
  └─ 레포 import 후 자동 배포 (10분)
```

---

## 박교수님의 방침 — 두 가지

### "포크는 하지 맙시다"

GitHub에는 **fork(포크)**라는 기능이 있습니다. 원본을 내 계정에 복사하는 기능인데, **이 자습서에서는 사용하지 않습니다**.

대신 **`git clone` 받아서 새 레포 만들고, 그쪽으로 push**합니다. 이유는 fork가 원본 레포와 영구 연결되어 나중에 충돌·동기화가 복잡해지기 때문. 7장에서 자세히 다룹니다.

### "쉽게 가르치자"

CMD 명령어 한 줄로 끝나는 일은 GUI(Graphical User Interface) 도구를 쓰지 않습니다. 그것이 결국 학생에게 더 빠르고 확실한 길이기 때문.

---

## 제2부 진행 중 자주 만나는 함정 8가지

봉주가 직접 겪었거나, 학생들이 자주 막힌 부분입니다. 본문에서 만날 때 [참고] 박스로 다시 강조됩니다.

| # | 함정 | 어디서 등장 |
|---|---|---|
| 1 | 드래그&드롭으로 아바타 파일을 옮기다가 엉뚱한 폴더에 떨어뜨림 | 6장 |
| 2 | GitHub 새 레포 만들 때 "Add README" 체크해서 충돌 발생 | 7장 |
| 3 | `git status`에서 untracked 파일이 보이는데 그게 뭔지 모름 | 7장 |
| 4 | Git 첫 push 시 인증 화면이 떠서 당황 | 7장 |
| 5 | `npm install`이 1~3분 멈춘 것처럼 보여 Ctrl+C 누름 | 5장 |
| 6 | `npm run dev` 창을 닫아서 봇이 죽음 | 5장 |
| 7 | 아바타 추가 후 일반 새로고침(F5)만 해서 캐시 때문에 안 보임 | 6장 |
| 8 | 아바타 파일을 `src/`에 넣어서 봇이 못 찾음 | 6장 |

---

## 제2부를 마치면

```
☐ C:\dev\my-bot 폴더에 스타터킷이 받아져 있음
☐ 내 컴퓨터에서 localhost:5173 으로 봇이 뜸
☐ 화면에 내 아바타가 움직이고 있음
☐ 내 GitHub 계정에 봇 레포가 올라가 있음
☐ Vercel에서 https://...vercel.app URL 발급됨
☐ 친구가 그 URL을 누르면 봇이 뜸
```

모두 ✅이면 1차 자습서 완주입니다.
