# 제1부. 준비물과 도구

> 봇을 만들기 전에 갖춰야 할 도구·계정·파일을 준비합니다. 한 번 갖춰두면 이후 어떤 코드 프로젝트에서도 재활용할 수 있습니다.

---

## 제1부의 목표

본격적인 실습(제2부) 전에 **세 가지 그릇**을 준비합니다.

1. **계정 그릇** — GitHub(코드 저장소), Vercel(인터넷 배포) 두 개의 무료 계정
2. **공간 그릇** — `C:\dev`라는 깔끔한 작업 폴더
3. **도구 그릇** — Node.js(자바스크립트 실행기)와 Git(버전 관리)

여기까지 갖춰두시면, 제2부의 봇 실습은 거의 명령어 복사·붙여넣기만으로 진행됩니다.

---

## 제1부의 구성

```
1장. 시작하기 전에 갖춰야 할 것
  └─ 계정 가입 + 다운로드 받기 (15~20분)

2장. 작업 공간 만들기 (C:\dev)
  └─ 폴더 하나 만들기 (5분)

3장. Node.js와 Git — 왜 둘 다 필요한가
  └─ 도구 설치 확인 (5~15분)
```

---

## 왜 이 순서인가

박교수님께서 강조하시는 학습 순서가 있습니다.

> "도구를 먼저 손에 익히게 하라. 코드를 보여주기 전에."

처음 코드를 만져보는 학생이 가장 자주 막히는 곳은 **본 실습이 아니라 준비 단계**입니다. "git이 뭐냐", "node가 왜 필요하냐", "VRM 파일이 어디 있냐" 같은 질문에서 막히면 본 실습 시작도 못 하고 끝납니다.

그래서 제1부에서 **천천히 모든 준비물을 갖춰두는 것**이, 결국 전체 시간을 줄이는 가장 빠른 길입니다.

> 📌 **[참고] "이미 다 깔려있는 것 같은데, 이 부분 건너뛰면 안 되나요?"**
>
> 됩니다 — 단, **3장 끝의 검증 표**는 반드시 확인하세요. `git --version`·`node --version`·`npm --version` 세 줄이 모두 정상이면 4장으로 가셔도 됩니다. 하나라도 안 되면 1~3장을 처음부터.

---

## 제1부를 마치면

다음 체크리스트가 모두 ✅이면 제2부로 가셔도 됩니다.

```
☐ GitHub 계정 만들었음 (https://github.com/본인ID 접속 가능)
☐ Vercel 계정 만들었음 (https://vercel.com/dashboard 접속 가능)
☐ VRM 파일 (.vrm) 바탕화면에 있음
☐ C:\dev 폴더 존재함
☐ git --version 명령어가 정상 작동함
☐ node --version, npm --version 명령어가 정상 작동함
```
